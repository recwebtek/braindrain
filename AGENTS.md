## Environment Context Protocol

Before running any shell commands, installing packages, or assuming tool
availability — call `get_env_context()` first.

This returns a cached OS fingerprint with zero token overhead after the first
probe. It tells you:
- Exact hostname, username, and LAN IPs (no guessing `sudo` user vs login user)
- Which package manager to use (`brew` vs `apt` vs `dnf` etc.)
- Which modern CLI tools are installed (`fd`, `bat`, `rg`, `fzf`...)
- Active runtimes and their version managers (`nvm`, `pyenv`, etc.)
- Shell type and config path
- Agent behaviour hints (what to prefer and what to avoid on this OS)

**Never probe the environment yourself** (no `which`, `command -v`, `uname`
chains). The cached context is authoritative. If something seems missing,
call `refresh_env_context()` and the user will confirm.
