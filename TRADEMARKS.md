## Braindrain Name and Branding Policy

This policy applies to the project name **Braindrain**, any official project logos, and related brand identifiers.

### Code license vs. brand usage

The source code is licensed under `AGPL-3.0` in `[LICENSE](LICENSE)`.
That license governs code use, modification, and distribution.
This policy is only about avoiding confusion around the official project identity.

### Community-friendly allowed uses

- Truthfully describing compatibility or origin (for example: "works with Braindrain", "forked from Braindrain", or "Braindrain-compatible").
- Community discussion, tutorials, blog posts, package metadata, and educational content that refer to the project accurately.
- Public forks that keep historical attribution and clearly indicate they are unofficial or modified versions.
- Internal/private use of the name when not presented as an official distributed product or service.

### Uses that need permission

- Presenting a product or hosted service as the official "Braindrain" offering when it is not.
- Using official Braindrain logos or branding in a way that implies endorsement, partnership, sponsorship, or maintainership.
- Registering domains, package names, or social accounts that are likely to confuse users about what is official.

### Forks and commercial use

Commercial use of the code is permitted under AGPL-3.0 compliance.
Commercial and non-commercial forks are welcome.
If you ship a fork or service, please use distinct branding unless you have explicit permission to present it as official Braindrain.

### Permission requests

If you want to use the Braindrain name or branding in a way that may look official, open an issue in this repository with the intended use and context.
